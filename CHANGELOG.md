## 提交规范

为了自动生成更新日志，请遵循以下提交信息格式规范：

- feat: 新功能
- fix: 修复问题
- docs: 文档变更
- style: 代码格式（不影响代码运行的变动）
- refactor: 重构（既不是新增功能，也不是修改bug的代码变动）
- perf: 性能优化
- test: 增加测试
- chore: 构建过程或辅助工具的变动

示例：
```
feat: 添加贪吃蛇移动功能

- 实现通过键盘控制蛇的移动
- 添加碰撞检测
- 实现得分统计
```
